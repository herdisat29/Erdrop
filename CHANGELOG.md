# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2026-06-12

### Added
- **Global Error Boundaries**: Introduced robust error boundaries at root and dashboard levels using Material Design 3 semantic classes.
- **Loading Skeletons**: Added UI loading skeletons for Analytics, Calendar, and Plan pages to ensure a smooth transition and reduce layout shifts.
- **Vercel Analytics & Speed Insights**: Integrated real-time tracking to monitor application performance and traffic right from day one.
- **Premium SEO Metadata**: Overhauled Open Graph and Twitter card tags to ensure professional link previews across social media (Telegram, X).
- **AI Import Framework**: Users can now import CSV data to be mapped via Gemini 2.5 Flash automatically.
- **AI Masterplans**: Dynamic 4-week task plans generated by analyzing user's active airdrops.
- **Cron Jobs**: Setup hourly cron API routes for sending automated email notifications using Resend.
- **Complete Dashboard**: Features trending coins, trending NFTs, and CryptoPanic news feeds.

### Changed
- Re-architected project detail layouts to solve overlapping elements, utilizing flexbox over absolute positioning.
- Completely redesigned the Analytics grid and Portfolio charts for full responsiveness without overlapping items.
- Upgraded the documentation (`README.md`) for proper setup guides.
- Upgraded overall typography consistency across headings and labels to align with `Plus Jakarta Sans`.

### Fixed
- Fixed an IDOR vulnerability in `/api/analyze/route.ts` where users could potentially access AI analysis data for projects they did not own.
- Addressed a TypeScript typing issue involving possibly `null` properties in the Resend instance initialization for NFT Cron emails.
- Fixed hydration mismatches by wrapping time/date objects in `<span suppressHydrationWarning>`.
- Resolved a conflict with Base UI (`@base-ui/react/menu`) by replacing `onSelect` with standardized `onClick` events inside `DropdownMenuItem` lists, preventing accidental router push events on parent Cards.

### Security
- Enforced `user_id` verification logic across all server actions mutating database objects (Projects, Logs, Plans, AI Analyses).
- Set up Supabase RLS bypass strictly controlled by Server Action boundaries utilizing Privy Authentication tokens.
